Theory

Introduction to Line Coding

Line Coding is the process of converting digital data (binary sequences of 0s and 1s) into digital signals suitable for transmission over a communication channel. The choice of line coding scheme affects bandwidth requirements, synchronization capability, error detection, and DC component of the signal.We need line coding techniques because raw digital data cannot be sent directly; it must be shaped into a suitable signal. Line coding ensures reliable transmission, maintains synchronization between the transmitter and receiver, reduces errors and noise, optimizes bandwidth usage, and in some cases even supports error detection. All these factors make digital communication accurate and robust.

Classification of Line Coding Schemes

1. Unipolar NRZ (Non-Return to Zero)

In Unipolar NRZ, the signal uses only one polarity (positive voltage). Logic 1 is represented by a positive voltage (+V), and logic 0 is represented by zero voltage (0V). The signal does not return to zero between consecutive bits.

Characteristics:

  • Simple implementation
  • Presence of DC component
  • No self-synchronization capability
  • Baseline wander problem for long sequences of 0s or 1s

2. Unipolar RZ (Return to Zero)

Similar to Unipolar NRZ, but the signal returns to zero in the middle of each bit period. Logic 1 is represented by +V for the first half and 0V for the second half. Logic 0 remains at 0V throughout.

Characteristics:

  • Better synchronization than NRZ due to transitions
  • Requires more bandwidth (twice that of NRZ)
  • Still has DC component issues

3. Polar NRZ

Uses both positive and negative voltage levels. Logic 1 is represented by +V and logic 0 by -V. This is more efficient than unipolar schemes.

Characteristics:

  • Better noise immunity than unipolar
  • No DC component for balanced data
  • More power efficient
  • Synchronization issues for long runs of same bit

4. Polar RZ

Combines the advantages of polar signaling with return-to-zero. Signal returns to zero in the middle of each bit period.

Characteristics:

  • Better synchronization than Polar NRZ
  • Requires twice the bandwidth
  • Good noise immunity

5. Bipolar AMI (Alternate Mark Inversion)

Logic 0 is represented by zero voltage. Logic 1 alternates between +V and -V. Consecutive 1s have opposite polarity.

Characteristics:

  • No DC component
  • Simple error detection (violation of alternation rule)
  • Better bandwidth utilization
  • Problem with long sequences of zeros

6. Pseudo Ternary

Pseudo Ternary is a modified encoding scheme where the roles of 1s and 0s are reversed compared to AMI. Binary 1 is represented by 0V (zero voltage), while binary 0 alternates between positive voltage (+V) and negative voltage (-V).

Logic:

  • Binary 1: 0V (No pulse)
  • Binary 0: Alternates between +V and -V

Characteristics:

  • Uses three voltage levels (+V, 0V, -V)
  • No DC component for balanced data
  • Good for long sequences of 1s (represented as no pulse)
  • Better synchronization for data with many 0s
  • Inverse representation of Bipolar AMI

7. Manchester Encoding

Also known as bi-phase encoding. Logic 1 is represented by a transition from low to high in the middle of the bit period. Logic 0 is represented by a transition from high to low.

Characteristics:

  • Excellent self-synchronization (transition in every bit)
  • No DC component
  • Used in Ethernet (10BASE-T)
  • Requires twice the bandwidth

8. HDB3 (High Density Bipolar of order 3)

An improvement over AMI that prevents long sequences of zeros. When four consecutive zeros occur, they are replaced by a special code that includes a violation (same polarity as previous 1).

Characteristics:

  • Solves the long zero sequence problem of AMI
  • Maintains synchronization
  • No DC component
  • Used in E1 carrier systems
  • Case 1: Even number of pulses since last violation → use B00V
  • Case 2: Odd number of pulses since last violation → use 000V

9. B8ZS (Bipolar with 8-Zero Substitution)

A line coding method used mainly in North America to solve the problem of long sequences of zeros in AMI. When eight consecutive zeros (00000000) occur, they are replaced with a special pattern 000VB0BV (appearing inside the 8-zero block), where V represents a bipolar violation and B represents a balancing pulse.

Substitution Pattern:

The 8 zeros 00000000 are replaced with: 000VB0BV
• B = Balancing pulse (same polarity as expected alternate pulse)
• V = Violation pulse (same polarity as previous pulse)

Characteristics:

  • Replaces long sequences of 8 zeros
  • The pattern 000VB0BV maintains synchronization
  • Has no DC component
  • Widely used in T1 carrier systems

Step-by-Step Procedure

Phase I: System Initialization

  1. Path Activation: Click functional blocks in chronological order:
    • Input Signal → Line Coder → Virtual Oscilloscope
    • Input Signal → Line Coder → Power Spectral Density (PSD) Analyzer
  2. Initialization Check: Confirm that the waveform and PSD display area become visible after completing the block sequence.The PSD will only appear when the Line Encoder block is connected to the PSD block.

Phase II: Binary Input Configuration

  1. Enter Binary Sequence: In the Binary Input field, type your desired 8-bit binary sequence (e.g., 10110100).
  2. Validation Check: Ensure only 0s and 1s are used. A default sequence is available for testing.
  3. Generate Signal: Click the GENERATE button to display the waveform.
  4. Select Encoding Scheme: Use channel selector buttons to switch between:
    • Input – Original digital signal
    • Output – Unipolar NRZ, Unipolar RZ, Polar NRZ, Polar RZ, Bipolar AMI, Manchester, HDB3, Pseudo Ternary
  5. Comparrison of Output Signals: Evaluate and compare the output signals of each encoding scheme.

Phase III: Control Parameter Adjustment

  1. Adjust Bit Rate: Control the number of displayed bits (4–32 bits). Pattern repeats automatically.
  2. Adjust Amplitude: Set encoded signal voltage (1V–10V). Input signal amplitude remains fixed at 5V.

Phase IV: PSD Analysis & Observation

  1. Analyse the Power Spectral Density (PSD) output:
    • Theory vs simulation results
    • Change in bit rate
    • Frequency components

Results and Observations

Observations for Each Line Coding Scheme

1. Unipolar NRZ

Voltage Levels Used: 0V and +V

Observations:

  • Uses only positive voltage levels
  • Simple implementation
  • Presence of DC component
  • No synchronization for long sequences of same bit

2. Unipolar RZ

Voltage Levels Used: 0V and +V (returns to 0)

Observations:

  • Signal returns to zero in middle of bit period
  • Better synchronization than NRZ
  • Requires double bandwidth

3. Polar NRZ

Voltage Levels Used: +V and -V

Observations:

  • Uses both positive and negative voltages
  • Better noise immunity than unipolar
  • No DC component for balanced data
  • More power efficient

4. Polar RZ

Voltage Levels Used: +V, 0V, and -V

Observations:

  • Returns to zero between bits
  • Good synchronization capability
  • Higher bandwidth requirement

5. Bipolar AMI

Voltage Levels Used: +V, 0V, and -V (alternating for 1s)

Observations:

  • Logic 1s alternate between positive and negative
  • No DC component
  • Simple error detection capability
  • Problem with long sequences of zeros

6. Pseudo Ternary

Voltage Levels Used: +V, 0V, and -V (alternating for 1s)

Observations:

  • Logic 0s alternate between positive and negative
  • Logic 1s are represented by 0V (no pulse)
  • No DC component
  • Similar to AMI, but 1s and 0s are swapped
  • Used where 1s occur more frequently than 0s

7. Manchester Encoding

Voltage Levels Used: +V and -V (with mid-bit transitions)

Observations:

  • Transition in middle of every bit period
  • Excellent self-synchronization
  • No DC component
  • Requires double bandwidth
  • Used in Ethernet networks

8. HDB3

Voltage Levels Used: +V, 0V, and -V (with violation codes)

Observations:

  • Prevents long sequences of zeros
  • Uses violation codes for four consecutive zeros
  • Maintains synchronization
  • No DC component
  • Used in E1 telecommunication systems

9. B8ZS

Voltage Levels Used: +V, 0V, and -V (with substitution codes)

Special substitution 000VB0BV when 8 zeros occur

Observations:

  • Substitutes eight consecutive zeros with a special pattern
  • Maintains synchronization
  • No DC component
  • Introduces B (balancing pulse) and V (violation pulse)
